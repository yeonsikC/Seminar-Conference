# 1. 전통적 자연언어처리 기법과 문제점 - 지식 중심 접근
- 인공지능 연구의 양대 흐름
  * 지식 중심 접근 vs 데이터 중심 접근
- 지식 중심 접근(규칙 기반 접근)
  * 1980년대 인공지능 연구의 주된 흐름
  * 전문가(인간)로부터 추출한 규칙/사실 정보를 코드/데이터에 반영
  * 작은 저장용량, 느린 처리 속도에 알맞은 현실적인 접근 방법
- 데이터 중심 접근
  * 문제 처리에 필요한 입력의 분포 특성을 프로그램이 데이터로부터 스스로 학습
  * 2010년대 : 통계 기반 접근에서 딥러닝 기반 접근으로 중심 이동
  * 거대한 저장용량, 빠른 연산 속도 덕분에 가능해진 접근 방법
# 전통적 자연언어처리 기법과 문제점 - 단계별 처리
- 자연언어처리의 주요 단계
  * 음성처리, 어휘처리, 구문처리, 의미처리
- 단계별로 다양한 규칙 및 사전 정보 활용
  * 음운모델, 어휘규칙, 어휘사전, 문법규칙, 대화 모델, 도메인 지식, 세계 지식 ...
- 모호성
  * 각 단계 처리 결과는 나름의 모호성을 갖는다
  * 각 단계의 모호성이 결합할 때 전체 모호성은 지수 함수적으로 증가
# 전통적 자연언어처리 기법과 문제점 - 현황
- 지식 중심 접근 방법의 어려움
  * 전문가 지식 추출의 어려움 : 지속적 확장이 쉽지 않음
- 지식 중심 접근 방법에서 자연언어처리의 난제들
  * 방대한 사전 정보의 수집과 정제
- 자연언어처리 실용화 현황
  * 검색 등 주요 어휘처리 수준에서 실용화가 이루어짐
# 전통적 자연언어처리 기법과 문제점 - 말뭉치 구축
- 전통적 기법에서의 말뭉치의 필요성
  * 어휘 정보 수집의 원천 자료
- 데이터 중심 접근 방법에서의 말뭉치의 필요성
  * 딥러닝 알고리즘을 위한 학습 데이터 : 구칙 및 어휘 정보의 ㅏㅈ동 학습
  * 다양한 용도에 맞게 레이블링 처리된 대량의 말뭉치 필요
- 말뭉치 구축 현황
  * 세종말뭉치(2007~2011) : 국립국어원 언어정보나눔터(원시 3,700만어절 + 가공)
  * 2차 세종 말뭉치 사업(2018~2022) : 총 154억 7천만 어절의 말뭉치 구축 중
# 2. 딥러닝을 이용한 자연언어처리 기법 - 딥러닝 모델
- 2010년대 이후 크게 각광받고 있는 데이터 중심 접근 방법
  * 퍼셉트론 신경망을 이용하여 입력 분포 특성을 자동 학습
- 딥러닝 모델의 강점
  * 데이터로부터의 자동 학습 : 전문가의 지식 추출 과정 불필요
  * 모순된 지식이나 결여된 지식으로 인한 오동작으로부터 강건
  * 보다 폭넓은 문맥 요소 고려해 처리 가능
  * 모델에 알맞은 출력 생성 용이, 데이터 친화성 높아 멀티모달 처리도 용이
  * 강화학습에 의한 지속적 성능 향상 가능
# 2. - 단어에서 벡터로
-- word2vec
- 언어 처리 과정에서 단어를 관련 정보 벡터로 대치해 처리
  * 단어 알파벳 혹은 단어 식별자 수준의 정보 : 딥러닝 처리에 턱없이 빈약
  * 전통적인 사전 정보는 단어의 벡터 정보로 활용하기 부적합
    -> 말뭉치에서 주변의 단어 분포를 일종의 맥락 삼아 조사한 후 통계 처리
    -> CBOW 모델 : 통계 처리를 딥러닝 학습으로 대치
- CBOW 모델에서 어휘수 증가에 따른 계산량 폭주 방지책
  * Embedding 계층의 구현 : 원-핫 벡터 처리를 효율적 대치
  * 네가티브 샘플링 : 정답 외의 부정적 답을 제한, 객관식 문제 풀듯이 학습 진행
- SKIP-GRAM 모델 : CBOW 모델의 입출력을 뒤바꾸어 변형
# 2 - CBOW 모델과 SKIP-GRAM 모델
